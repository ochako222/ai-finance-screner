# Implementation Plan: Currency Fix, USD Conversion & Dashboard Layout

**Branch**: `001-fix-currency-ui-charts` | **Date**: 2026-06-01 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/001-fix-currency-ui-charts/spec.md`

## Summary

Fix all currency symbol bugs across the app (T212 data is PLN, not USD), add a live PLN→USD conversion using the NBP exchange rate fetched during sync, introduce an Investment Dynamics area chart (total PLN over time), and restructure the dashboard into a four-row layout: totals cards → charts row → T212 full-width table → Binance full-width table.

**Language/Version**: TypeScript 5 (ESM), Node.js 22 LTS (server + build)

**Primary Dependencies**:
- Server: Fastify 5, better-sqlite3 12, undici 7, smol-toml
- Client: React 19, Vite 6, Chart.js 4 + react-chartjs-2 5, Zustand 5, React Query 5

**Storage**: SQLite at `~/.local/share/alex-financial-screener/portfolio.db` (better-sqlite3, no ORM)

**Testing**: No test suite in project; Biome lint must pass before commit

**Target Platform**: Local desktop — single user, `localhost:7788`

**Project Type**: Local web application (Fastify API + React SPA)

**Performance Goals**: Dashboard loads from SQLite cache within 200 ms; dynamics chart renders within 1 s; sync completes within 10 s (dominated by T212 rate-limited calls)

**Constraints**: T212 rate limits (1 req/5s account, 1 req/1s positions); exchange rate fetch is non-blocking / non-fatal; all colors from `_variables.scss`; no inline `any` in client code

**Scale/Scope**: Single user; ≤ 100 positions; ≤ 365 history points per year

## Implementation Tasks

### Task 1 — Database: add `portfolio_history` table and functions

**File**: `src/server/database.ts`

1. In `getDb()`, add the `portfolio_history` table creation to the `exec` block:
   ```sql
   CREATE TABLE IF NOT EXISTS portfolio_history (
       id            INTEGER PRIMARY KEY AUTOINCREMENT,
       captured_at   TEXT NOT NULL,
       total_pln     REAL NOT NULL,
       total_usd     REAL,
       exchange_rate REAL
   );
   CREATE INDEX IF NOT EXISTS idx_history_time ON portfolio_history (captured_at DESC);
   ```

2. Add `savePortfolioHistory(totalPln, totalUsd, exchangeRate)` function — inserts a row with `new Date().toISOString()`.

3. Add `loadPortfolioHistory()` function — returns all rows ordered by `captured_at ASC`, mapped to `PortfolioHistoryPoint[]`.

4. Update `loadLatestSnapshot()` return type: rename `totalUsd` → `totalPln`, add `totalUsd: number | null` and `exchangeRate: number | null`. These come from the latest `portfolio_history` row matching the same `captured_at` (or left-join pattern using the most recent history row).

**Simple approach for step 4**: Read the most recent `portfolio_history` row separately and merge into the return value. No JOIN needed since both are always written together.

### Task 2 — Server: exchange rate connector

**File**: `src/server/connectors/exchangeRate.ts` (new)

```ts
// Fetch today's PLN/USD rate from NBP.
// Returns PLN per 1 USD (e.g., 4.02).
// Falls back to last available rate if today's is not yet published.
// Throws if both attempts fail.
export async function fetchPlnUsdRate(): Promise<number>
```

Implementation notes:
- Primary: `GET https://api.nbp.pl/api/exchangerates/rates/a/usd/today/?format=json`
- Fallback: `GET https://api.nbp.pl/api/exchangerates/rates/a/usd/?format=json&last=1`
- Parse `response.rates[0].mid` as the rate (PLN per 1 USD).
- Use `undici`'s `fetch` (already available via Node 22 globals or `undici` import).
- Log errors with `fastify.log` pattern — but this is a standalone connector, so use `console.error` minimally and let the caller handle the error.

### Task 10 — Component: new DynamicsChart

**File**: `src/client/components/DynamicsChart.tsx` (new)

```tsx

import type { PortfolioHistoryPoint } from '../support/types';
```

Props: `{ history: PortfolioHistoryPoint[] }`

Behavior:
- If `history.length === 0`: render empty state with stub-notice text "No sync history yet."
- If `history.length === 1`: render single-point chart + subtitle "Sync regularly to see your portfolio trend."
- Otherwise: render area line chart with dates on x-axis (short format: `DD MMM`) and PLN totals on y-axis.

Chart.js options:
- `fill: true` with gradient fill using `$accent` → transparent
- Line color: `$accent` (`#667eea`)
- Point color: `$accent`
- Grid color: `$border` (`#30363d`)
- Tick color: `$text-muted` (`#8b949e`)
- Y-axis formatter: `toLocaleString('pl-PL', { style: 'currency', currency: 'PLN', maximumFractionDigits: 0 })`
- X-axis: short date label — `new Date(pt.capturedAt).toLocaleDateString('pl-PL', { day: '2-digit', month: 'short' })`

Data fetching: `DynamicsChart` receives `history` as a prop from `DashboardPage`, which fetches it via `usePortfolioHistory` hook (React Query).
